Simulated Garbage Collector

Custom allocator (duMalloc) and Java-style GC concepts in C with managed pointers and generational collection.

This project simulates a managed runtime in C across four versions of a custom allocator. It progresses from classic heap allocation strategies to a managed-pointer layer and finally a generational collector with minor (copying) and major (in-place) collections.

  • V1: first-fit / best-fit allocation + memory dumps
  • V2: managed pointers (indirection table) & pointer tracking
  • V3: young generation + copying collector (minor GC)
  • V4: old generation + compaction & survival tracking (major GC)

  • Version 1 — First-fit / best-fit allocator with dumps.
    Outputs: FIRST · BEST
  • Version 2 — Managed pointer layer & indirection.
    Output: run log
  • Version 3 — Young gen with copying minor GC.
    Output: run log
  • Version 4 — Old gen with compaction & survival tracking (major GC).
    Output: run log

  • Language: C (GCC)
  • Env: VS Code, WSL
  • Debug: Valgrind, printf dumps
  • Concepts: first/best-fit, freelists, managed pointers, copying minor GC, major GC with compaction
